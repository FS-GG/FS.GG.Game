# `template/` — FS.GG.Game producer content

The producer-side template content FS.GG.Game owns after the ADR-0022 extraction.

## `product-skills/`

The four **game product skills** migrated from FS.GG.Rendering in **ADR-0022 P4** (skill-ownership
`fs-gg-rendering → fs-gg-game`): `fs-gg-game-core`, `fs-gg-audio`, `fs-gg-persistence`,
`fs-gg-model-swap`. Each `SKILL.md` is **byte-identical** to the copy FS.GG.Rendering still ships
from its **frozen** `dotnet new fs-gg-ui --profile game` (the accepted two-copies cost during the
extraction — ADR-0022 §6; retired by the P6 provider epic).

## `skill-manifest/skill-manifest.json`

The **producer skill-manifest** (schema v1) declaring those four skills — the contract the org
registry `registry/skills.yml` (FS-GG/.github) reconciles its `owner: fs-gg-game` rows from, so
**registry = manifest = bytes**. Generated by `scripts/generate-skill-manifest.fsx` (recomputes each
`SKILL.md` digest); a CI gate (`gate.yml` → *Skill-manifest drift*) fails on any drift.
`materializes-when` is written in the ADR-0017 **canonical** grammar the skill-union gate evaluates.

## Not here yet

The `dotnet new fs-gg-game` **template package**, its `.template.config/template.json`, the game
**fragments** (collision/grids/line-drawing/visibility — still frozen in Rendering), and the
consumer **scaffold provider** are deferred to the sequel provider epic (ADR-0022 §2.1 "later" / P6).
This directory is content + manifest only; nothing here is wired to a scaffold yet.
